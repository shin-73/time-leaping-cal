# Time-Leap Cal 品質保証基準

この文書は、現行 `spec.md` を唯一の評価基準として、ナラティブ品質とアプリ挙動を検証するためのQA基準である。

## 1. ナラティブ品質（五感同期と没入感）

- [ ] 歴史事実の箇条書きではなく、情景描写を伴う物語として読める。
- [ ] 五感描写（視覚・聴覚・触覚・嗅覚）のうち、複数要素が含まれている。
- [ ] 文体は `PROMPT_TEMPLATE` の制約に沿い、説明的な定型文から開始しない。
- [ ] APIエラー時は `${year}年（${eraName}）の記憶にアクセスできませんでした。` が表示される。

## 2. UI/UX演出（タイピング実装）

- [ ] 生成本文は全文即時表示ではなく、一文字ずつ段階的に表示される。
- [ ] 句点（。！？）・読点（、）・改行で待機時間が変化し、等速表示になっていない。
- [ ] `prefers-reduced-motion` 有効時はタイピング演出を抑制し、即時表示へ切り替わる。
- [ ] 句点（。！？）ごとに段落化され、段落間に余白がある。
- [ ] 年齢文は本文と同一エリア・同一スタイルで、本文表示後に続けて表示される。
- [ ] 年齢文パートは本文より遅い速度設定（通常文字 `x3.0625`、読点 `x2.25`）が適用される。
- [ ] タイピングスキップUI（`Skip` ボタン等）が表示されない。

## 3. 機能整合性（入力・年号・連動）

- [ ] 入力は西暦/和暦の各モードで `spec.md` の活性条件どおりに遷移する。
- [ ] 全角数字入力は半角へ正規化され、数字以外の文字が除去される。
- [ ] 入力は最大4桁に制限される。
- [ ] 1868年未満はエラー表示となり、結果表示へ遷移しない。
- [ ] 遷移後の表示年と和暦変換結果が入力内容と一致する。
- [ ] URLパラメータ（`year`, `era`）が状態を保持し、戻る操作で画面状態が復元される。

## 4. エラーハンドリングと耐障害性

- [ ] Gemini未設定時にユーザーへ失敗理由が表示される。
- [ ] API失敗時（Timeout含む）はエラー文言を表示し、アプリがクラッシュしない。
- [ ] API失敗時は年齢文を表示しない。
- [ ] API成功時はナラティブがセッション内メモリキャッシュから再利用される。
- [ ] 60秒タイムアウト設定で応答待ちが制御される。
- [ ] 再試行可能エラー時はモデルごとに1回の自動リトライが実行される。
- [ ] モデル候補（`gemini-1.5-flash` → `gemini-flash-latest`）のフォールバック試行が機能する。

## 5. ビジュアル・レイアウト安定性

- [ ] Hero画像は横幅いっぱいに表示され、年号の可読性が保たれる。
- [ ] トップページと結果画面で年号表示の位置とスケールが破綻しない。
- [ ] モバイルでテンキー操作後も画面崩れや主要UIの押下不能が発生しない。
- [ ] ページ遷移時に先頭へスクロールされ、閲覧開始位置が安定する。

## 6. テスト運用ルール

- [ ] 仕様変更時は、まず `spec.md` を更新し、同時に本基準も差分同期する。
- [ ] QAは「ナラティブ品質」「UI演出」「機能整合」「エラー時挙動」を最低1ケースずつ実施する。
- [ ] 不一致を検出した場合、コード修正前に `spec.md` との差分を明文化して合意を取る。
