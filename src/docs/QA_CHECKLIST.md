# Time-Leap Cal：マスター・テストケース（最終正解定義）
**MASTER_PRD.md v1.3 と常に同期すること。**

---

### 1. トップページ仕様（SP版の引き算と安定性）
- [ ] **SP版ボタンの不在**: スマホ版（sm未満）のトップページに「タイムリープ」ボタンは存在しない。YEAR入力フォームのみ。
- [ ] **SP版操作**: 4桁入力後、スマホキーボードの「確定/Go/開く」で結果画面へ遷移する。
- [ ] **iOS実機テスト**: iOSテンキー上の「完了（Done）」ボタン押下時、`onBlur` ロジックによって正しく結果画面へ遷移する。
- [ ] **PC版ボタン**: PC版（sm以上）で、4桁入力時のみ活性化する「タイムリープ」ボタンが表示・動作する。
- [ ] **Y軸不動テスト**: PC版（1200px幅）で、遷移前後の年号要素の `getBoundingClientRect().top` が **316px** で完全に一致し、1pxのガタつきも発生しない。
- [ ] **横ズレ防止**: スクロールバーの有無に関わらず、年号の左右座標が動かない（`scrollbar-gutter: stable`）。
- [ ] **レイアウト安定性**: キーボードの開閉を行っても、`min-h-[100dvh]` と `fixed` ヘッダーにより、画面がズレたり「？」等のアイコンがクリック不能にならない。
### 2. ナラティブの品質（文体の聖典）
- [ ] **文体**: 「だ・である」調と「体言止め」による硬質なアーカイブ記録となっており、抽象的な形容詞（「懐かしい」「美しい」等）が含まれていない。
- [ ] **3大要素の網羅**: 1.マクロな事件、2.ミクロな生活感、3.現在との対比・断絶の3要素が順に構成されている。
- [ ] **事実ベース**: 抽象的な表現ではなく、歴史的事実に基づいた密度が確保されている。
- [ ] **文字数**: 150〜200文字程度のボリュームが確保されている。
- [ ] **固有名詞**: 具体的な名称が3つ以上含まれている。
- [ ] **フォールバック品質**: APIエラー時は事実のみ（「アクセスできませんでした」）が表示される。

### 3. 年齢表示（AGE_PHRASES定数より）
- [ ] **誕生前**: 「あなたがこの世に生を受ける、〇年前の出来事だった。」
- [ ] **誕生年**: 「それは、あなたがこの世に生を受けた、記念すべき年だった。」
- [ ] **誕生後**: 「あなたはこの時、〇歳だった。」
- [ ] **地続き表示**: 年齢フレーズが歴史テキストと同一スタイル（`block mb-6`）で地続きに表示される。

### 4. 結果画面の構成
- [ ] **最小主義**: メインエリアはAI生成文のみ。誕生日設定ボタン等のCTAは存在しない。

### 5. 入力バリデーション（不変のガードレール）
- [ ] **4桁制限**: すべてのYEAR入力欄で5桁以上の入力が物理的に不可能。
- [ ] **明治以前バリデーション**: 1868年未満の入力に対し、適切なエラーが表示され遷移がブロックされる。
- [ ] **エラー表示位置**: エラーメッセージが `position: absolute` で完全にフローティング表示され、出現しても入力欄や他のUI要素の座標（Y軸等）を1ピクセルも押し下げない。
- [ ] **文字種制限**: 数字以外の文字（e, +, -, 記号等）は入力不可。

### 6. ビジュアル・レイアウト（Ultimate Flat）
- [ ] **Hero全幅**: 画像が左右の端まで1ピクセルの隙間もなく表示される。
- [ ] **和暦の美的調整**: 和暦テキストが白と黒の微細なシャドウによって「金属板に打たれた刻印（レタープレス）」のような品格を持たせている。
- [ ] **画像オーバーレイ**: Hero画像のオーバーレイが極薄（alpha 0.2〜0.3程度）であり、画像の質感を損なわずに文字の可読性を確保している。
- [ ] **直角デザイン**: アプリ内の全要素に角丸が適用されていない（CSSグローバルリセットで強制）。

### 7. オンボーディング（全画面ポスター化）
- [ ] **SP版全画面**: `fixed inset-0` で余白・角丸なしの完全全画面表示。
- [ ] **文言**: 読点（、）での改行なし、句点ごとのまとまり、`space-y-6` の行間。
- [ ] **ヘルプ**: ヘッダーの「？」アイコンからいつでも再表示できる。
